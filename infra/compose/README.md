O proxy reverso pode ser definido de duas formas: usando o Nginx diretamente ou usando annotations do Nginx no Ingress do Kubernetes. A escolha pelo Ingres do Kubernetes é mais eficiente e simples para realizar configurações, independendo de conceitos e detalhes específicos do Nginx e de controle de redes. Para acessar a documentação das Annotations do Nginx, acesse aqui [Nginx Ingress Annotations](https://kubernetes.github.io/ingress-nginx/user-guide/nginx-configuration/annotations/).
